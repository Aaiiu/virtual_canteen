# 虚拟食堂仿真系统 - 需求文档

## 1. 项目背景
学生食堂在用餐高峰期经常出现排队时间长、找座位难的问题。本系统通过模拟和可视化技术，帮助学生合理安排就餐时间。

## 2. 功能需求

### 2.1 核心功能（P0）

#### 2.1.1 窗口菜系设置
- 每个打饭窗口设置具体菜系（如天津菜、湘菜、汉堡快餐、川菜、清真等）
- 不同菜系设置不同的基础就餐时间
- 负责人：成员A

#### 2.1.2 学生属性与特性词条系统
- 每个学生具备以下属性：
  - 菜品偏好（对应某一菜系窗口）
  - 个体速度系数（0.8~1.2，影响就餐时长）
  - 特性词条（随机分配，0至多个）
- 特性词条库（存储于 config.py，可扩展）：
  - 选择恐惧症：打饭时间+50%
  - 急性子：强制选择等待时间最短的窗口
  - 社恐：优先选择排队人数最少的窗口
  - 孤独症：只坐没有人坐过的座位
- 超参数：可调节各特性词条在人群中的触发比例
- 负责人：成员B

#### 2.1.3 排队模拟系统
- 模拟多窗口排队场景
- 排队决策逻辑：
  - 学生优先前往偏好菜系窗口排队
  - 若偏好窗口等待时间超过设定阈值，切换至等待时间最短的窗口
  - 特性词条可覆盖以上默认决策（如急性子强制选最短等待）
- 模拟学生到达率（高峰期12:00-12:30人流量高，其余时段较低）
- 负责人：成员B

#### 2.1.4 就餐时间系统
- 不同菜系设置不同基础就餐时间：
  - 快餐类：5分钟
  - 普通菜：10分钟
  - 复杂菜：15分钟
- 最终就餐时间 = 基础时间 × 个体速度系数
- 特性词条进一步修正（如选择恐惧症+50%，急性子-20%）
- 负责人：成员A + 成员B

#### 2.1.5 座位管理系统
- 座位分配：普通学生选择最近空闲座位
- 孤独症特性：只选择从未被坐过的座位
- 座位释放：就餐结束后自动释放
- 负责人：成员A

#### 2.1.6 2D食堂可视化
- 绘制食堂平面图（含各菜系窗口标签、座位区域）
- 实时显示各窗口排队人数
- 实时显示座位占用状态（颜色区分空闲/占用）
- 鼠标悬停已占用座位时显示浮窗：
  - 学生姓名、学号
  - 就餐菜品偏好
  - 特性词条列表
  - 就餐开始时间
  - 剩余就餐时间
- 负责人：成员C

### 2.2 重要功能（P1）

#### 2.2.1 模拟对比功能
- 操作者可设置菜品偏好参数
- 系统分别运行两种策略：
  - 策略A：学生按菜品偏好选择窗口
  - 策略B：学生按等待时间最短选择窗口
- 输出对比数据：
  - 等待时间对比
  - 寻座时间对比
  - 就餐时间对比
  - 总耗时对比
- 负责人：成员B + 成员C

#### 2.2.2 时间段分析
- 统计不同时间段的平均等待时间
- 标注高峰时段
- 生成可视化图表
- 负责人：成员B

#### 2.2.3 最佳时间推荐
- 基于等待时间 + 座位可用性 + 用户偏好推荐最佳就餐时间
- 输出偏好菜系 vs 最短等待策略的建议对比
- 负责人：成员B

#### 2.2.4 数据可视化图表
- 等待时间折线图（含不同菜系窗口对比）
- 人流量柱状图
- 座位占用率饼图
- 特性人群分布图（各特性人群占比）
- 负责人：成员C

### 2.3 可选功能（P2）
1. 3D可视化升级
2. 特性词条扩展（8-10个词条）
3. 数据导出功能（CSV、图表导出）

## 3. 非功能需求
- 响应时间：页面加载 < 2秒
- 兼容性：支持主流浏览器（Chrome、Edge）
- 可维护性：代码注释完整，结构清晰
- 演示性：系统运行稳定，演示数据准备充分

## 4. 数据库设计概览

| 表名 | 主要字段 | 说明 |
|------|----------|------|
| windows | id, name, cuisine, service_time | 窗口菜系信息 |
| students | id, name, preference, speed_factor | 学生基本属性 |
| student_traits | student_id, trait_name | 学生特性词条关联 |
| traits | name, probability, effect_param | 特性词条库 |
| seats | id, status, student_id, ever_used | 座位状态 |
| queues | window_id, count, wait_time | 实时队列信息 |
| simulation_logs | time, window_id, wait_time | 模拟历史记录 |
