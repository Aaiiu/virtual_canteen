## 二、安装 Python

### Windows

1. 访问 [Python 官网](https://www.python.org/downloads/) 下载 Python 3.10 安装包（`.exe`）

```bash

# 安装 Python
brew install python@3.10

# 验证
python3 --version
pip3 --version
```

## 四、创建虚拟环境（推荐）

虚拟环境可以让每个项目拥有独立的依赖包，避免版本冲突。**强烈建议**在虚拟环境中开发。

```bash
# 在项目根目录下执行（virtual_canteen/ 目录内）

# 创建虚拟环境，命名为 venv
python -m venv venv

# 激活虚拟环境
## Windows（命令提示符 / PowerShell）
venv\Scripts\activate

## macOS / Linux
source venv/bin/activate
```

激活成功后，命令行提示符前会出现 `(venv)` 字样，例如：

```
(venv) C:\Users\你的用户名\virtual_canteen>
```

> ⚠️ **每次开发前**都需要重新激活虚拟环境。退出虚拟环境使用命令：`deactivate`

## 六、初始化数据库

项目使用 SQLite 数据库，数据库文件会自动生成在 `data/` 目录下（文件名：`canteen.db`）。

首次运行前，需要执行数据库初始化脚本，创建表结构并插入测试数据：

```bash
# 在项目根目录下执行
python scripts/init_database.py
```

执行成功后，`data/canteen.db` 文件会被创建。

> ℹ️ SQLite 是一种轻量级数据库，**无需单独安装数据库服务器**，Python 内置支持，开箱即用。

## 十、常见问题 FAQ

### Q1：`python` 命令不存在或提示找不到？

**原因**：Python 未添加到系统 PATH。  
**解决**：
- Windows：重新运行 Python 安装程序，勾选 `Add Python to PATH`，或手动添加环境变量
- macOS：使用 `python3` 代替 `python`

---

### Q2：`pip install` 报错 `Could not find a version that satisfies the requirement`？

**原因**：网络问题或 pip 版本过低。  
**解决**：
```bash
# 升级 pip
python -m pip install --upgrade pip

# 使用清华镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

---

### Q3：启动 `app.py` 报错 `ModuleNotFoundError: No module named 'flask'`？

**原因**：虚拟环境未激活，或依赖未安装到当前环境。  
**解决**：
```bash
# 确认虚拟环境已激活（命令行前有 (venv) 字样）
venv\Scripts\activate       # Windows
source venv/bin/activate    # macOS / Linux

# 重新安装依赖
pip install -r requirements.txt
```

---

### Q4：浏览器访问 `localhost:5000` 显示「无法访问此网站」？

**原因**：Flask 服务器未启动，或端口被占用。  
**解决**：
1. 确认终端中 `python app.py` 正在运行，且无报错
2. 若端口被占用，修改 `src/app.py` 最后一行的端口号：
```python
# 将 5000 改为其他端口，如 5001
app.run(debug=True, host='0.0.0.0', port=5001)
```
然后访问 `http://localhost:5001`

---

### Q5：`python scripts/init_database.py` 报错？

**原因**：脚本尚在开发中，或数据库路径不存在。  
**解决**：
1. 确认 `data/` 目录存在（若不存在，手动创建该文件夹）
2. 检查 `config/config.py` 中 `DATABASE_PATH` 路径配置是否正确

---

### Q6：Git 克隆时提示权限错误？

**原因**：未配置 Git 用户信息或 SSH 密钥。  
**解决**：
```bash
# 配置 Git 用户信息（首次使用必须配置）
git config --global user.name "你的姓名"
git config --global user.email "你的邮箱@example.com"
```

## 附录：项目关键配置参数

配置文件位于 `config/config.py`，开发时可根据需要调整：

```python
class Config:
    # 数据库路径（相对于 src/ 目录）
    DATABASE_PATH = '../data/canteen.db'

    # 食堂基础配置
    WINDOW_COUNT = 8    # 打饭窗口数量
    SEAT_COUNT   = 200  # 座位总数

    # 模拟参数
    SIMULATION_SPEED = 1.0             # 模拟速度倍率（1.0 = 实时）
    PEAK_HOURS = [(11, 13), (17, 19)]  # 高峰时段（小时）
```
